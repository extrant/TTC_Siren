# TTC_Siren

面向《最终幻想 XIV》九宫幻卡的自动对战求解器。项目提供一个本地 Flask AI 服务，将客户端识别到的棋盘、手牌、规则信息转换为可执行的出牌建议。

当前核心目标不是给出“看起来分高”的一步棋，而是在隐藏手牌、加算/同数、同类强化、残局反制等复杂局面下尽量减少误判。

## 核心能力

- **Minimax + Alpha-Beta 搜索**：主路径使用迭代加深、置换表、历史启发和走法排序。
- **信息集残局求解**：空位较少且未知牌不超过 2 张时，构造多个可能世界并完整搜索到终局。
- **合法未知牌枚举**：残局优先从全卡库枚举符合当前已知信息的候选，排除已出现卡牌，并应用卡组限制。
- **隐藏手牌建模**：前中局使用规则相关采样，高风险局面会扩大对手候选池。
- **规则适配评估**：支持基础翻面、同数、加算、逆转、王牌杀手、同类强化/弱化、秩序、混乱等规则。
- **服务端实时速度显示**：控制台输出搜索深度、节点数、NPS、置换表命中率、剪枝率和残局信息集进度。
- **HTTP 接口集成**：客户端只需 POST 当前局面，即可获取推荐卡牌和放置位置。

## 支持的规则

当前代码重点支持：

| 规则 | 状态 |
| --- | --- |
| 基础大小翻面 | 支持 |
| 同数 / 加算 | 支持，特殊翻面后才触发连携 |
| 逆转 | 支持 |
| 王牌杀手 | 支持，且仅处理 1 与 A(10) 的特殊克制 |
| 同类强化 / 同类弱化 | 支持，场上同类型卡牌数量决定修正层数 |
| 全明牌 / 三明牌 | 支持输入侧明牌/未知牌建模 |
| 秩序 / 混乱 | 支持 `canUse` 手牌可用性 |
| 选拔 | 有采样建模，仍建议结合实战验证 |

卡组限制按官方规则处理：**★4 以上卡牌合计不超过 2 张，★5 卡牌最多 1 张**。

## 求解流程

1. 服务端解析客户端传入的棋盘、我方手牌、对手手牌、当前玩家和规则。
2. 已知卡牌通过四边数值匹配数据库 ID，未知牌用 `0,0,0,0` 表示。
3. 前中局使用 Minimax 搜索，并对未知牌进行规则相关采样。
4. 进入残局时，如果满足空位较少且未知牌不超过 2 张：
   - 从全卡库枚举合法未知牌候选。
   - 按当前棋盘威胁排序并构造有限数量的信息集场景。
   - 对每个我方候选走法在所有场景中搜索到终局。
   - 按安全比例、最坏收益和综合收益覆盖标准搜索结果。
5. 返回推荐卡牌、位置、对手手牌分析、胜率估计和性能统计。

## 快速开始

### 环境要求

- Python 3.8+
- Windows / macOS / Linux
- 依赖：`flask`, `pandas`, `numpy`

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动服务

```bash
python ai_server.py
```

默认监听：

```text
http://127.0.0.1:5000
```

启动后，服务端窗口会显示实时搜索速度，例如：

```text
[Search] depth=8 nodes=144,248 nps=15,169 tt=24.5% cutoff=34.2% elapsed=9.51s
[Endgame] move=6/12 scenario=15/24 nodes=135 nps=269 elapsed=0.50s
```

## HTTP 接口

### `POST /ai_move`

请求体示例：

```json
{
  "board": [
    {
      "pos": [0, 0],
      "numU": 7,
      "numR": 4,
      "numD": 9,
      "numL": 4,
      "owner": 1
    }
  ],
  "myHand": [
    { "numU": 7, "numR": 6, "numD": 2, "numL": 3, "canUse": true }
  ],
  "oppHand": [
    { "numU": 0, "numR": 0, "numD": 0, "numL": 0, "canUse": true }
  ],
  "myOwner": 1,
  "currentPlayer": 1,
  "rules": "三明牌, 加算",
  "solver": "minimax"
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `board` | 当前棋盘卡牌列表，空格不需要传 |
| `pos` | 棋盘坐标 `[row, col]`，范围 `0..2` |
| `owner` | `1=蓝方`，`2=红方` |
| `myHand` | 我方剩余手牌 |
| `oppHand` | 对手剩余手牌，未知牌四边都填 `0` |
| `myOwner` | 我方颜色，`1=蓝方`，`2=红方` |
| `currentPlayer` | 当前行动方，`1=蓝方`，`2=红方` |
| `rules` | 规则文本，可包含多个中文规则名 |
| `solver` | `minimax` 或 `monte_carlo`，默认 `minimax` |

响应会包含推荐卡牌、落点、胜率估计、对手手牌分析和性能统计。

## 测试

运行回归测试：

```bash
python -m pytest -q
```

当前测试覆盖重点：

- 未知手牌槽位数量不膨胀。
- 同数/加算翻面和连携判定。
- 同类强化/弱化修正层数。
- 王牌杀手与逆转交互。
- 残局信息集安全选择。
- 合法未知牌候选的星级限制过滤。

## 当前限制

- 主 Minimax 仍是单进程搜索；实时 NPS 可观察性能，但尚未使用多进程并行根节点。
- 未知牌信息集会从合法候选池中按风险截断，避免全组合爆炸；它不是无上限的全卡库笛卡尔积搜索。
- 胜率和推荐文案是辅助信息，最终可信度以实际棋盘规则执行和终局搜索结果为准。
- 客户端识别错误会直接影响 AI 判断，尤其是 owner、当前回合、`canUse` 和规则文本。
